Hyphe
Hyphe is a web corpus curation tool
featuring a research-driven web crawler

Hyphe provides users with a method to build web corpora

In Hyphe a web corpus is a set of web pages curated and organized by the user

Granularity: group web pages

as “web entities”

In Hyphe, you decide how web pages have to be grouped. You can group them by URL: when they share the same domain.tld, or the same subdomain.domain.tld, or the same subdomain.domain.tld/path or…

it’s up to you

Web pages groups are called “web entities”. This concept is more flexible and precise than the intuitive but vague notion of “website”.

Thanks to the web entities, your web corpus will have the granularity you need, and the web pages will be organized as you want

Curation: step by step

Corpus expansion

In Hyphe, you decide if a given web entity should be included or excluded into your corpus: you are a web corpus curator and as such

you define the borders of your own corpus

Hyphe uses a web crawler that never harvests out of web entities you specifically targeted. It will retrieve internal pages of a given web entity (down to a defined depth) but will never follow outbounding links.

You decide

The crawler prospects potential new web entities to be included and you decide what to do with these (include, exclude, crawl...)

Quality first

This curation activity takes time, often a lot of time. On a speed vs. quality tradeoff, we invested on quality having librarians and social scientists as target

Flexibility:

Dynamic aggregation

In Hyphe, the definition of web entity boundaries can be changed at any time. Hyphe features a flexible memory structure which gives you the ability to

change your mind on the way

Hyphe’s memory structure is built on a 3-levels data aggregation system.

Level 0: Raw data

Crawled web contents are stored as down loaded. Level 0 can be rebuilt by re-crawling

Level 1: Web pages

Raw data are indexed at the web page level. Level 1 can be rebuilt (at a higher computational price)

Level 2: Web entity

Web page level is dynamically aggregated to the web entity level. Level 2 is dynamically built from level 1

Hyphe curation principles are implemented as a HTML5 User Interface.

Hyphe’s UI guides you through each step of the web corpus curation method and helps you monitoring your collect.

Javascript

enabled crawler

Hyphe includes a javascript enabled crawler using

PhantomJS

You can decide to use it systematically (though it is much slow er) or to try without javascript by default and automatically retry failed crawls with javascript enabled.

Hundreds of thousands of web pages

Tens of thousands of web entities

Hyphe targets relatively small web corpora. It is about curated web corpora, not big data.

We want performance!

The memory structure is based on efficient Lucene indexes but no sharding

Hyphe behaves nicely with hundreds of thousands crawled web pages (and a billion detected) grouped in tens of thousands of web entities for a 2Go RAM and 5Go file system footprint

I want it!

Hyphe is packaged for recent Ubuntu, Debian and Cent OS.

Do you prefer an API?

You don't like our method or user interface but want to use the crawler and memory structure in a different way? You can learn how to use the core API or develop your own UI.
